hugo+Github搭建博客

300 Words|Read in about 2 Min|Views:

纠结了很久，最后还是选择把Blog转到Hugo上来，由于怕麻烦就不把以前的Hexo迁移到Hugo上了，所以整了个新的Hugo，力求简洁。按照惯例，既然捣鼓了Hugo，首篇Blog就自然介绍一下Hugo+Github搭建博客吧！

环境

  • Windows 10
  • Git

安装Hugo

选择对应的版本下载Hugo

新建hugo目录将压缩包解压至bin目录下：

解压完成后，在此目录下打开cmd，输入hugo version可见详细版本信息，即代表安装成功；为了可以在任意目录下运行hugo命令，建议添加到Path环境变量中。

C:\Users>hugo version
Hugo Static Site Generator v0.62.0-6608F155 windows/amd64 BuildDate: 2019-12-23T10:26:18Z
C:\Users>

生成site目录

进入hugo目录

hugo new site blog

#Just a few more steps and you're ready to go:
#
#1. Download a theme into the same-named folder.

# 目录结构
tree blog
#blog
#├── archetypes
#│   └── default.md
#├── config.toml
#├── content
#├── data
#├── layouts
#├── static
#└── themes

config.toml 是配置文件，在里面可以定义博客地址、构建配置、标题、导航栏、主题等等。

themes 是主题目录，可以去https://themes.gohugo.io/ 下载喜欢的主题。

content是博客文章的目录。

安装主题

下面用 hello-friend-ng 举例

  1. 下载

    可以直接clone到themes目录下，有点是如果对主题有调整可以同时提交到git控制中。

    $ git clone https://github.com/rhazdon/hugo-theme-hello-friend-ng.git themes/hello-friend-ng

  2. 使用

    直接复制该主题中config.toml中的内容，覆盖hugo目录下config.toml文件中的内容

第一篇文章

在hugo\blog目录下

hugo new blog/20200105/hugo搭建博客.md

文章顶部可以设置一些meta信息，例如：

---
title: "xxx"
date: 2020-01-04T00:30:32+08:00
weight: 70
keywords: ["hugo"]
description: "第一篇文章"
tags: ["hugo", "pages"]
categories: ["pages"]
author: ""
---

这里是文章内容

顶部的meta信息是默认生成的，当然也可以手动添加，如果不想每次都手动添加，可以到archetypes目录下的default.md中修改模板。

本地预览

执行命令，使用Hugo生成静态网页内容并在本地启动HTTP Server。然后即可访问http://localhost:1313/查看效果。

hugo server --theme=hello-friend-ng --buildDrafts -w

–theme 指定应用的主题

–buildDrafts 发布时包含草稿也一并部署，文章顶部draft=true时生效

-w 监视本地文件的改动，实时更新，无需重新运行hugo server命令

部署到Github Page

首先需要创建自己的Github Page，新建仓库repository，命名为username.github.io 注意

username一定要与你github的名称一致。

创建完成后，进入仓库，点击setting，滑到Github Pages并选择一个主题（任意就行）

接着，用Hugo生成静态网页，站点目录下hugo\blog\，cmd输入命令：

hugo --theme=hello-friend-ng --baseUrl="https://Sun-GoKu.github.io/"

执行完上面的命令后，在站点目录下会生成一个public文件夹，我们只需将此文件夹内的所有内容推上仓库就可以了。进入public的cmd目录，鼠标右键Git Bash Here，（第一次执行git init后此后就不需要执行这句命令了）

具体的Git操作是：

git init
git add .
git commit -m "first commit of blog"
git remote add origin https://github.com/Sun-GoKu/Sun-GoKu.github.io.git
git pull --rebase origin master
git push -u origin master

其中，如果是第一次创建仓库，git pull --rebase origin master此条命令报错则不用管，直接运行最后一条命令push到远程仓库去。(这里如果对git不熟悉的话，建议去学一下廖雪峰老师的Git教程)

push成功后，查看https://Sun-GoKu.github.io/ ：

至此，hugo+Github Page搭建博客基本完成。
